Today in Lab we are going to go over how to sum the nodes in a tree of a generic type. The purpose of this lab is to write an algorithm that sums up all the nodes in the Tree and returns back the integer representation of it. This can be tricky when you plan to use the + operator to sum up every value. You cannot use + because the Tree provided to you is of type T extends Comparable <T> & OperatorOverload<T> . What this means is that the Tree contains data of type T which in turn implements the Comparable and the Operator Overload interface 

public String getSum() throws EmptyTreeException {
        if (root==null)
            throw new EmptyTreeException();
        return _getSum(root).toString();

    }

The return type of getSum is String and not int. Why is this? This is because when you add up integers you get an int. However, when you add up values of type T you get back a value of type T. So by making the return type String, we simply make our method more flexible such that the Tree returns back the String representation of T. If T were an Integer, you get back the String representation of Integer. If T were a CustomIntegerForLab, you get back String representation of CustomIntegerForLab etc etc. This is really really cool!
