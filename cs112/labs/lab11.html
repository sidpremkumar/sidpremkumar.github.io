What is an iterator?

An iterator is an object that enables a programmer to traverse a container, generally a list. In the classic case, an iterator starts at the first element of a list. Then, it moves forward one element at a time when the method next() is called. In the most simple sense, it is another technique for going through the items in a data structure. You can learn more about iterators here. For information about Java- specific iterators, check here.

PreOrder

In both BinaryTree and Tree, you are expected to implement an iterator that goes over the tree in PreOrder. Here is a quick refresher on PreOrder:

BinaryTree.java

In BinaryTree.java, when a tree is constructed, BinaryTreeIterator creates a Java stack named s. Immediately, the root of the tree is pushed onto the stack. Then, left and right children are added to the root.
Your job is to implement the hasNext and next methods of BinaryTreeIterator. You will ”iterate” over the list in preOrder by pushing and popping items from the stack in the correct order. Remember, stacks are a first-in, last-out data structure! Here are some guidelines for the methods you need to implement:

  1. hasNext()

    2. You can implement this by simply checking whether there is anything on the stack. If the stack is empty, there are no more nodes to iterate over.

  2. next()
    • Call hasNext() to make sure there is another node to iterate over. If there isn’t, return null.
    • If there is another node, pop it off of s and save it.
    • Now push each of the children onto the stack s. If you want to iterate over nodes in preOrder, which child should you push first: the right or the left? Think carefully about this!
    • Return the data from the node you popped.

Tree.java

Similar to the previous example, when a tree is constructed in Tree.java, TreeIterator creates a Java stack named s. Again, the root of the tree is pushed onto the stack. However, since we have a general tree structure, children are added and stored in a linked list.
Here are some guidelines for the methods you need to implement:

  1. hasNext()

    2. Once again, to implement this method, simply check if stack s is empty.

  2. next()
    • Call hasNext() to make sure there is another node to iterate over. If there isn’t, return null.
    • If there is another node, pop it off of s and save it.
    • Now push each of the children onto the stack s. This time, since there can be a variable amount of children, they are all stored in a linked list. In which order do we want to push items from the linked list onto the stack? Think carefully about this! (Hint: you might find the Java method LinkedList.descendingIterator very useful. Here is a link to its documentation).
    • Return the data from the node you popped.
