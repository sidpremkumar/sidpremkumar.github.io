What is a Binary Search Tree?

A BST is a binary tree in which each node has a key that holds a value. The nodes have the following properties:

  1. The left subtree of a node contains only nodes with keys less than the nodes key.
  2. The right subtree of a node contains only nodes with keys greater than the nodes key.
  3. Both the left and right subtrees of a node must also be binary search trees.
  4. For this lab, we can assume there are no duplicate nodes. (As you implement the methods in this lab, think about why this makes things easier. On your homework, you will explore some strategies for dealing with duplicate nodes in BSTs.)

Insertion

How does insertion work in a BST?
A new key is always inserted as a leaf, which is a node without any children. We start searching from the root, and we compare our key to the current node at each step along the way. Eventually, we will we find a leaf node, and we will add our key as its child. Here is an example:

Search

How does search work in a BST?
We are given a key, and our goal is to find out if the key appears in the BST. If it does, we return true. Otherwise, we return false. To begin, we compare our key with the root of the tree. If they are the same, we can return true right away. However, if the key is less than the value of the root, we recurse over the left subtree. Likewise, if the key is larger than the value of the root, we recurse over the right subtree.

InOrder Trasversal

How does InOrder Traversal work in a BST?
Everything in the left subtree of the root is less than the root, and everything in the right subtree is greater than the root. Therefore, an InOrder traversal of a BST automatically puts all of the elements in order!
